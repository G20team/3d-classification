## 環境情報

### Python環境

#### **パッケージ管理**: uv

`uv run` を基本とし、対話シェルが必要な場合のみ `source .venv/bin/activate` を使ってください。

**注意**: `uv run` を使うとコマンド単位で環境が適用され、待機状態を回避できます。

**推奨**: `uv sync` 後に `uv run <command>` を使ってください。

```bash
# 推奨されない方法
source .venv/bin/activate  # 必要な場合のみ

# 推奨される方法
uv run python script.py
uv add package_name && uv sync
```

#### スタイル

スタイルの設定は`.vscode/settings.json`を参照してください。これらに違反するような書き方は避けてください。
警告が表示された場合は、警告内容に従ってコードを修正することを徹底するようにしてください。

ワークスペースの特性上、ファイルpathを使用することが多くなりますが、管理が煩雑になることを防ぐため、ハードコーディングされたパスは使用しないでください。パスの管理を一元化する仕組みを提案してください。

Linterとしては`ruff`が使用されています。設定は`pyproject.toml`を参照してください。
`ruff`の警告が表示された場合は、警告内容に従ってコードを修正することを徹底するようにしてください。

Pythonコードでは型ヒントを積極的に活用し、新規関数・メソッドでは引数と戻り値の型注釈を付与してください。
特に公開API・入出力境界・設定処理では、`Any` の濫用を避けて型を明示してください。

大きな変更を加えた場合は、最低限のテストを実行して、コードが正しく動作することを確認してください。（テストの方法は基本的にはpytestになります。修正内容によってはpytestの追加や修正も検討してください。）

### コーディング指針

関数・クラス・ファイルは、コンポーネントごとに分割することで、再利用性を高め、適切な粒度を保つようにしてください。

#### ファイル構造パターン

```sh
project/
├── configs/          # YAML設定ファイル
├── data/             # データセット
├── src/              # ソースコード
│   ├── preprocessing/  # データ前処理
│   ├── ./       # 適宜必要なディレクトリをコンポーネントごとに作成
│   └── utils/          # ユーティリティ
├── scripts/         # 実行スクリプト
├── outputs/         # 実験結果
└── notebooks/       # Jupyter Notebook
```

#### 重要な実装ポイント

- **設定管理**: YAML設定ファイルによるパラメータ管理
- **ログ管理**: 実験の再現性のためにログやconfigのsnapshotを残すようにしてください（tensorboardを用いて訓練の進捗を追跡できるようにしてください）
- **エラーハンドリング**: データ破損・形式不正への対応
